左值引用笔记

Posted on 2025-09-01 Edited on 2025-09-02 In 编程入门 , C++

基础概念

名称 说明
左值（Lvalue） 在内存中有持久存储地址的对象，有名字，可取地址。如int x=5;中的 x
右值（Rvalue） 临时对象、字面值、表达式结果，无法直接取地址。如 5, x+1, std::string(“abc”) 返回的匿名对象
右值引用（T&&） C++11 新增的引用类型，可以绑定到右值上，实现资源“转移”（移动语义）

右值引用的主要作用

  1. 实现移动语义（Move Semantics）
    避免不必要的拷贝，将资源从临时对象转移
  1. 完美转发（Perfect Forwarding）
    搭配 std::forward 和万能引用实现泛型函数参数完美传递：
  1. 延长临时对象生命周期
    右值引用变量绑定临时对象后，会延长其生命周期

右值引用 T&& 绑定右值，用于移动语义
万能引用（Forwarding Reference） template void f(T&& x) 推导时可同时绑定左值和右值

std::move和std::forward

函数 作用
std::move 将对象显式转换为右值引用，触发移动构造或移动赋值。注意不会真的移动资源，只是“允许移动”
std::forward 条件性地将参数转发为右值或左值，用于完美转发模板

示例：

移动语义使用场景

  • 容器类（std::vector, std::string 等）在扩容时，使用移动构造避免深拷贝
  • 返回局部对象时，编译器启用 RVO（Return Value Optimization），减少拷贝
  • 大对象传递时，可以通过 std::move 将临时对象直接移动

编译器优化相关（RVO/NRVO）

  • C++17 起，返回局部对象几乎必定省略一次移动或拷贝
  • std::move有时会关闭 RVO，要谨慎：

注意事项

  1. 移动后对象仍然有效，但状态未定义
    只能安全销毁、赋值、或重新初始化，不能假设内容还在
  2. 小对象移动收益小
    移动语义的意义主要是避免昂贵的深拷贝
  3. 右值引用不能绑定左值
    必须 std::move 或 std::forward 才能绑定左值
  4. 传值参数和移动语义
    函数参数按值传递时，即使传入右值，也会先调用拷贝/移动构造函数创建副本

附录

std::move标准实现（简化版本）

核心逻辑：

  1. 模板参数 T
    T 是通过传入对象类型推导出来的，T&& 是万能引用
  2. remove_reference
    去掉类型中的引用，让 T&& 最终成为一个真正的右值引用类型
  3. static_cast<…&&>
    用 static_cast 把传入对象转换为右值引用

使用示例：

std::move(x) 不会移动对象内容，只是产生一个右值引用，允许你调用移动构造函数或移动赋值运算符

std::forward标准实现（简化版本）

核心逻辑：

  1. std::forward 实际上和 std::move 一样，都是 static_cast<T&&>(arg)
  2. 区别在于：
    • std::move 总是无条件转换成右值引用
    • std::forward 会根据模板参数 T 来决定是否转换成右值引用
  3. std::forward 的重载 + static_assert 保证：
    • 如果 T 是 U&，返回的是左值引用 U&
    • 如果 T 是 U（非引用），返回的是右值引用 U&&

模板参数推导规则

对于函数模板参数推导，规则[1]是：

  • 如果模板参数是 T&&，而实参是左值：
    • T 推导为 左值引用类型，即 int&
  • 如果实参是右值：
    • T 推导为实参类型本身，即 int

type traits笔记

Posted on 2025-04-25 Edited on 2025-09-02 In 编程入门 , C++

type traits

type traits 是 C++ 标准库提供的一组模板类和模板变量，用于 在编译期查询类型特性、进行类型变换或静态断言

特点：

  • 编译期执行（constexpr）
  • 不需要运行时开销
  • 常用于泛型编程、模板元编程、完美转发等

类型属性查询

判断一个类型是否满足某种条件：

trait 功能 示例
std::is_integral 是否整型 std::is_integral::value → true
std::is_floating_point 是否浮点型 std::is_floating_point::value → true
std::is_pointer 是否指针 std::is_pointer<int*>::value → true
std::is_reference 是否引用 std::is_reference<int&>::value → true
std::is_lvalue_reference 是否左值引用 std::is_lvalue_reference<int&>::value → true
std::is_rvalue_reference 是否右值引用 std::is_rvalue_reference<int&&>::value → true
std::is_const 是否 const std::is_const::value → true
std::is_same<T,U> 两类型是否相同 std::is_same<int,int>::value → true

这些常用于 static_assert 或模板特化做约束

类型变换

在编译期生成新的类型：

trait 功能 示例
std::remove_reference 去掉引用 std::remove_reference<int&>::type → int
std::remove_const 去掉 const std::remove_const::type → int
std::add_lvalue_reference 添加左值引用 std::add_lvalue_reference::type → int&
std::add_rvalue_reference 添加右值引用 std::add_rvalue_reference::type → int&&
std::decay 类型衰减，去掉引用、const、数组转指针等 std::decay<int[3]>::type → int*

std::forward 就依赖 std::remove_reference 或 std::decay 来实现完美转发

类型属性组合与逻辑

C++17 引入 _v 后缀变量模板，便于逻辑组合：

这在模板 SFINAE 和 enable_if 中非常常用

SFINAE / 条件模板控制

std::enable_if 是 type traits 的经典应用：

  • std::enable_if_t<cond, T> = T if cond true; else 不存在 → SFINAE 生效

其他实用 traits

  • std::is_trivially_copyable：是否可以按位拷贝
  • std::is_default_constructible：是否可以默认构造
  • std::is_nothrow_move_constructible：移动构造是否不抛异常
  • std::alignment_of：类型对齐
  • std::extent：数组维度

使用场景

  1. 完美转发：配合 std::forward 和 std::remove_reference
  2. 静态断言：static_assert(std::is_integral::value, “必须整型”);
  3. SFINAE：std::enable_if 控制模板实例化
  4. 优化：区分 trivially copyable vs 非 trivially copyable
  5. 泛型算法：根据类型特性做不同处理

实现原理简析

C++ type traits 的实现本质上是 模板特化 + constexpr static 成员：

  • 通用模板给默认值
  • 偏特化匹配特定类型
  • C++17 引入 _v 变量模板简化 .value

fmtlog源码扫读版本

核心特点

  • SPSC（单生产者单消费者）无锁队列
    • 每个线程都有自己的ThreadBuffer（内部维护一个SPSCVarQueue）
    • 无锁操作，纳秒级别的操作性能
  • 日志格式化异步化
    • 日志格式化是慢操作（fmt::format是CPU密集型）
    • fmtlog会把格式化操作放到后台线程中异步执行
    • 前端只做简单的参数编码和推入队列
  • 后端线程的最小堆调度机制
    • 后台线程从所有线程的队列中取日志时，使用一个最小堆维护每个线程当前队头的时间戳
    • 确保日志的全局时间顺序正确，且能保证一定程度的均匀调度
  • 自定义时间戳采样器
    • 使用RDTSC指令获取时间戳，比std::chrono快得多
    • 用一个轻量的时间校准机制（tscns）将tsc映射为纳秒，取代慢速系统调用
  • 刷盘策略合理
    • 写文件时不强制fflush，除非：
      • 手动调用poll(true)
      • flush超时到了
    • 避免I/O瓶颈拖慢写入线程
  • 内存预分配：内部使用fmt::MemoryBuffer的对象缓存，避免频繁分配和释放内存
  • 不拷贝写入：参数编码也不需要格式化字符串那样的对象生命周期控制
  • 如果没有合适的回调处理，在队列满的场景下会丢日志

核心代码布局

  • fmtlog.h
    • 头文件只暴露接口和模板声明
  • fmtlog-inl.h
    • 包含所有实现细节，包括模板定义、inline函数实现等
  • fmtlog.cc
    • 通过#include "fmtlog-inl.h"把实现编译成一个.o文件，从而提供可选的非header-only使用方式
