左值引用笔记

Posted on 2025-09-01 In 编程入门 , C++

基础概念

名称 说明
左值（Lvalue） 在内存中有持久存储地址的对象，有名字，可取地址。如int x=5;中的 x
右值（Rvalue） 临时对象、字面值、表达式结果，无法直接取地址。如 5, x+1, std::string(“abc”) 返回的匿名对象
右值引用（T&&） C++11 新增的引用类型，可以绑定到右值上，实现资源“转移”（移动语义）

右值引用的主要作用

  1. 实现移动语义（Move Semantics）
    避免不必要的拷贝，将资源从临时对象转移
  1. 完美转发（Perfect Forwarding）

  1. 延长临时对象生命周期
    右值引用变量绑定临时对象后，会延长其生命周期

右值引用 T&& 绑定右值，用于移动语义
万能引用（Forwarding Reference） template void f(T&& x) 推导时可同时绑定左值和右值

std::move和std::forward

函数 作用
std::move 将对象显式转换为右值引用，触发移动构造或移动赋值。注意不会真的移动资源，只是“允许移动”
std::forward 条件性地将参数转发为右值或左值，用于完美转发模板

示例：

移动语义使用场景

  • 容器类（std::vector, std::string 等）在扩容时，使用移动构造避免深拷贝
  • 返回局部对象时，编译器启用 RVO（Return Value Optimization），减少拷贝
  • 大对象传递时，可以通过 std::move 将临时对象直接移动

编译器优化相关（RVO/NRVO）

  • C++17 起，返回局部对象几乎必定省略一次移动或拷贝

注意事项

  1. 移动后对象仍然有效，但状态未定义
    只能安全销毁、赋值、或重新初始化，不能假设内容还在
  2. 小对象移动收益小
    移动语义的意义主要是避免昂贵的深拷贝
  3. 右值引用不能绑定左值
    必须 std::move 或 std::forward 才能绑定左值
  4. 传值参数和移动语义
    函数参数按值传递时，即使传入右值，也会先调用拷贝/移动构造函数创建副本

std::move标准实现（简化版本）

核心逻辑：

  1. 模板参数 T
    T 是通过传入对象类型推导出来的，T&& 是万能引用
  2. remove_reference
    去掉类型中的引用，让 T&& 最终成为一个真正的右值引用类型
  3. static_cast<…&&>
    用 static_cast 把传入对象转换为右值引用

使用示例：

std::move(x) 不会移动对象内容，只是产生一个右值引用，允许你调用移动构造函数或移动赋值运算符

std::forward标准实现（简化版本）

核心逻辑：

  1. std::forward 实际上和 std::move 一样，都是 static_cast<T&&>(arg)
  2. 区别在于：
    • std::move 总是无条件转换成右值引用
    • std::forward 会根据模板参数 T 来决定是否转换成右值引用
  3. std::forward 的重载 + static_assert 保证：
    • 如果 T 是 U&，返回的是左值引用 U&
    • 如果 T 是 U（非引用），返回的是右值引用 U&&

模板参数推导规则

对于函数模板参数推导，规则是：

  • 如果模板参数是 T&&，而实参是左值：
    • T 推导为 左值引用类型，即 int&
  • 如果实参是右值：
    • T 推导为实参类型本身，即 int
